# 랩실 홈페이지 프로젝트 관리 계획서

## 1. 프로젝트 개요

본 프로젝트는 연구실을 소개하고 외부 사용자에게 연구 정보를 제공할 수 있는 **랩실 홈페이지**를 개발하는 것을 목표로 한다.

현재 연구실 정보가 문서, 메신저, 구두 안내 등으로 분산되어 있으면 연구실 소개가 어렵고, 외부 학생이나 협력 기관이 필요한 정보를 찾기 불편할 수 있다.

랩실 홈페이지는 이러한 문제를 해결하기 위해 연구실 소개, 연구 분야, 구성원, 프로젝트, 공지사항, 연락처 등의 정보를 한 곳에서 확인할 수 있도록 하는 웹 기반 시스템이다.

---

## 2. 프로젝트 목표

본 프로젝트의 주요 목표는 다음과 같다.

- 연구실의 소개와 비전을 한눈에 확인할 수 있도록 한다.
- 연구 분야와 진행 중인 프로젝트를 체계적으로 정리한다.
- 교수님, 연구원, 학생 등 구성원 정보를 제공한다.
- 연구실 관련 공지사항이나 소식을 확인할 수 있도록 한다.
- 외부 사용자가 연구실 위치와 연락처를 쉽게 확인할 수 있도록 한다.
- PC와 모바일 환경에서 모두 보기 쉬운 반응형 웹페이지를 구현한다.
- GitHub를 이용하여 문서와 소스코드를 관리한다.

---

## 3. 주요 기능

| 구분 | 기능 | 설명 |
|---|---|---|
| F01 | 메인 페이지 | 연구실 이름, 대표 이미지, 간단한 소개 문구를 제공한다. |
| F02 | 연구실 소개 | 연구실의 목표, 비전, 주요 연구 분야를 설명한다. |
| F03 | 구성원 소개 | 교수님, 연구원, 학생 등 구성원 정보를 제공한다. |
| F04 | 연구 분야 소개 | 연구실에서 다루는 주요 연구 주제를 정리한다. |
| F05 | 프로젝트 소개 | 진행 중인 프로젝트와 완료된 프로젝트 정보를 제공한다. |
| F06 | 공지사항 | 연구실 소식, 세미나, 모집 공고 등을 안내한다. |
| F07 | 연락처 | 이메일, 위치, GitHub 링크 등의 정보를 제공한다. |
| F08 | 반응형 화면 | PC, 태블릿, 모바일 화면에서도 사용할 수 있도록 한다. |

## 4. 개발 방식

본 프로젝트는 **애자일 기반의 반복 개발 방식**으로 진행한다.

처음부터 모든 페이지를 완성하기보다, 핵심 페이지를 먼저 제작한 뒤 필요한 내용을 점진적으로 추가한다.  
기능이나 디자인에 수정이 필요한 경우 GitHub issue 또는 commit 기록을 통해 변경 내용을 관리한다.

진행 방식은 다음과 같다.

1. 홈페이지에 필요한 정보를 정리한다.
2. 주요 페이지와 기능을 목록으로 작성한다.
3. 메인 페이지와 소개 페이지를 우선 구현한다.
4. 구성원, 프로젝트, 공지사항, 연락처 페이지를 순차적으로 구현한다.
5. 구현한 결과를 GitHub에 업로드한다.
6. 다른 사람의 문서를 확인하고 issue 또는 pull request로 리뷰를 남긴다.
7. 받은 리뷰를 반영하여 문서를 수정한다.

## 5. 작업 분해 구조

프로젝트는 다음과 같은 흐름으로 진행한다.

- **요구사항 분석**
  - 랩실 홈페이지에 필요한 정보와 기능을 정리한다.
  - 연구실 소개, 구성원 소개, 프로젝트 소개, 연락처 기능을 중심으로 요구사항을 정리한다.
  - PC와 모바일 환경에서 보기 편해야 한다는 비기능 요구사항도 확인한다.

- **설계**
  - 홈페이지의 전체 메뉴 구조를 설계한다.
  - 메인 페이지, 소개 페이지, 구성원 페이지, 프로젝트 페이지, 연락처 페이지의 화면 구성을 정리한다.
  - 각 페이지에 들어갈 텍스트, 이미지, 링크 정보를 정리한다.

- **구현**
  - HTML을 이용하여 웹페이지의 기본 구조를 작성한다.
  - CSS를 이용하여 글꼴, 색상, 배치 등 화면 디자인을 적용한다.
  - 필요한 경우 JavaScript를 이용하여 메뉴 이동, 버튼 동작 등의 간단한 기능을 구현한다.
  - 구현한 파일을 GitHub에 업로드한다.

- **테스트**
  - 각 페이지가 정상적으로 열리는지 확인한다.
  - 메뉴 링크가 올바르게 이동하는지 확인한다.
  - PC와 모바일 화면에서 레이아웃이 깨지지 않는지 확인한다.
  - 오탈자와 누락된 정보가 없는지 확인한다.

- **문서화**
  - 프로젝트 관리 계획서를 작성한다.
  - README에 프로젝트 소개와 실행 방법을 정리한다.
  - 다른 사람의 리뷰를 확인하고 필요한 내용을 문서에 반영한다.

## 8. 품질 관리

프로젝트 품질을 높이기 위해 다음과 같은 방식을 사용한다.

- GitHub를 이용하여 문서와 소스코드를 관리한다.
- 주요 수정 사항은 commit message로 기록한다.
- 기능 추가나 수정이 필요한 부분은 issue로 정리한다.
- 다른 사람의 프로젝트 관리 계획서를 확인하고 issue 또는 pull request로 리뷰를 남긴다.
- 다른 사람이 남긴 리뷰를 확인한 뒤 필요한 내용을 문서에 반영한다.
- 페이지 구현 후 직접 실행하여 화면이 정상적으로 보이는지 확인한다.
- PC와 모바일 화면에서 레이아웃이 깨지지 않는지 확인한다.
